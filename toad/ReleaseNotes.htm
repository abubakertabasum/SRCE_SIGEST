Introduction

These Release Notes are intended to provide an overview of new functionality and to list many of the fixes that have been implemented in this release of TOAD. The Release Notes are also intended to help you upgrade to the new version. For an introduction to TOAD, also see readme.txt.

TOAD 7.2 includes a Script Manager for managing your scripts, a Configure Tools window, from which you can configure TOAD to launch other applications, and an optional Task Bar that shows your open connections. The Schema Browser has 3 new tabs. The Queues and Queue Tables tabs are for use with Oracle Advanced Queuing. The Types tab, which shows Object Types and Collection Types, has taken over the functionality of the earlier Object Browser, which has been removed from TOAD. TOAD 7.2 is also the first version of TOAD to be tested on Oracle 9i. A Space Manager for collecting statistics on tablespace usage has been added to the DBA module. The Procedure Editor has been rewritten, and many fixes have been implemented for the Debugger, the Compare Schemas window, and the Generate Schema Script window, as well as in other places throughout TOAD. We believe that you will be very pleased with this new version.

Upgrade to the TOAD schema

A function has been added to the TOAD schema, which is created through toadprep.sql. Compile this function in the TOAD schema to speed up the Blocking locks tab in Kill/Trace:

CREATE OR REPLACE function TOAD.ObjectName(in_object_id in number) return
varchar
is
return_string varchar2(100);
begin
select OWNER||'.'||OBJECT_NAME
into return_string
from all_objects
where object_id = in_object_id;
return return_string;
end ObjectName;
/

Grant execute on TOAD.ObjectName to public;

Changes to the PLSQL parser script
In order to use the new syntax highlighting features for views and synonyms, some changes to plsqlscr.txt are necessary. In order to upgrade an existing plsqlscr.txt for the new features, add the following lines:

Below the line:
#define st_KEYWORD 26
Add the lines:
#define st_VIEWNAMES 27
#define st_SYNNAMES 28

Below the line:
#define ss_D4 203
Add the lines:
#define ss_D5 205
#define ss_D6 206

Under "%%language", change StartState to
StartState = [0, ss_D1, ss_D2, ss_D3, ss_D4, ss_D5, ss_D6]

Add the following to the list of EditableStyles
('View Names', st_VIEWNAMES),
('Synonyms', st_SYNNAMES),

Under "%%effects", add
st_VIEWNAMES [UpperCase] clOlive _DEFAULT_BACKGROUND
st_SYNNAMES [UpperCase] clOlive _DEFAULT_BACKGROUND

Under "%%kwtables", add
ForState [ss_D5] endswith _non_alpha_
( st_VIEWNAMES is 'SALES'
)

ForState [ss_D6] endswith _non_alpha_
( st_SYNNAMES is 'SYNS'
)

Under "%%map", add
st_VIEWNAMES st_VIEWNAMES
st_SYNNAMES st_SYNNAMES


TOAD Security
The TOAD Security Feature has new functionality. If you are upgrading to TOAD 7.2 and have already set up your security features in a previous version of TOAD, you can run an upgrade script so you do not have to recreate the security tables and reassign all of the privileges. You will need to open the TOADSECURITY_Update.SQL file in the SQL Editor and execute in the TOAD schema on the target instance. This will truncate the TOAD_FEATURES table and insert the new and / or relabeled features. After you run the upgrade script, you will need to go back into the TOAD Features Security window and make sure everyone has the correct privileges for TOAD 7.2. If you are upgrading from 7.1 to 7.2, then instead of running the upgrade script, you can simply run the following statement in order to upgrade:

INSERT INTO TOAD.TOAD_FEATURES ( NAME, DESCR ) VALUES ( 'CHAINED ROWS', 'DBA Menu, Chained Rows Window');

Oracle 9i Support

TOAD 7.2 has been fully tested on Oracle 9i. If TOAD supports a feature on an Oracle 8i database, then it is also expected to support that feature on an Oracle 9i database. However, support for most new features of Oracle 9i has not yet been added to TOAD.

Space Manager   There are two new tabs on the DBA | View Tablespace window: Space History and IO History. These tabs, along with the set of tables and views in the TOAD schema that populate the graphs on these tabs as well as the new Toad_Spaceman package in the TOAD schema, comprise what is called the TOAD Space Manager. The Space History tab provides a graph showing historical space usage of tablespaces and datafiles. The IO History tab shows a historical graph of various IO metrics, such as physical reads and writes. To begin using this new feature, one must first create the tables and job which drives the population of the tables. To do so:
  • connect as TOAD and open DBA | Tablespaces
  • click the toolbar button on the far left of either of the new tabs, whose hint reads 'Create/Recreate Space Manager Tables'. If you are not the TOAD user you will be prevented from doing this.
Clicking OK for first time setup creates the tables, views, SM package and job, and backfills the tables if specified.

Afterward, any user can then click the 'Edit Space Manager Settings' button to modify the job and history retention settings.
  • The collection status group box at the top gives you the status of the SM job and allows you to toggle the status.
  • The collection schedule controls operate like the Create Job window. The user can choose the frequency with which the SM job executes the TOAD_SPACEMAN.Collect() procedure.
  • The 'History Retention' group box specifies how much data should be kept in the SM tables. Whenever the SM job runs it first removes old data.
  • The 'Data Initialization' group box offers the user the choice of starting with empty tables or 'back filling' the tables with data showing increasing space usage over time given the current space utitlization as the start. This only applies to first time setup.

Caveats

TOAD making itself the active application
Some users have reported problems with TOAD making itself the active application while the user is working in another application while on Windows 2000. This problem is fixed with Windows 2000 Service Pack 2.
Schema definition files not backward-compatible
Compare Schemas and Generate Schema Script are not compatible with pre-7.2 .def files.
SQL*Loader and Oracle 8.0.x
Using "Watch Progress" with an 8.0.x version of SQL*Loader on Windows 2000 causes a 100% CPU usage. The recommended solution is to uncheck "Watch Progress" on the last screen of the SQL*Loader Wizard, so that the SQL*Loader is launched outside of TOAD.
Space Manager collects stats <= 1x per day
The TOAD Space Manager will not store statistics for a collection interval of more than once per day.
