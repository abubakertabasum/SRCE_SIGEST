Formatter Plus™ "5-Minute" Walk-Through

Thank you very much for installing Formatter Plus™. This document will instantly familiarize you with the key features and benefits of this powerful product from Quest Software.  If you are using a DEMO version of Formatter Plus, please note the DEMO Limitations listed at the bottom of this walk-through.

When you first open Formatter Plus™, you will be greeted with 3 major windows:

  • Project Browser Window - with a Sample Human Resources Project
  • Editing Window - displays code and formatted output
  • Results Window - Code Review, Syntax-Checking and Code Profiling results

Each of these windows work together to help you format, code review and analyze single files - or entire applications - with just a click! Let's begin:

Format a Folder

We'll begin by showing how Formatter Plus can quickly format a group of files to a standard format (we're using Formatter Plus' defaults for our example, but you can change these to match your in-house standards!).

  1. The Sample Project consists of 5 folders and a ReadMe text file. Click on the (+) sign next to the Salary History folder, and 4 files will appear. Hold down the Shift key and highlight all 4 files.
  2. With your cursor positioned over the highlighted files, click the right-mouse button, and select Format Files. Click Yes to create an archive. In the Results Window, the formatted files will be displayed. Double-click on the first file (salhist.spb) to view the formatted code in the Editing Window.

    Find in Project Files

Our next example illustrates the power of locating all occurrences of a string inside multiple files with just a few mouse clicks.  Let's say that you want to find how the variable "employee_id_in" is used throughout your code.  Note:  This function will only work on files WITHIN a project.  To search individual files not located in a project, click the binocular icon on the tool bar.

  1. First, double-click on the employee.spb file in the employee folder in the Project Browser Window.  The employee.spb file will open in the Editing Window.
  2. Scroll down to line 67 (line and column numbers are displayed in a lower right box on the status bar).  With your cursor positioned over the string "employee_id_in", right mouse click and select Find in Project Files.  
  3. The Results Window will display a list of all the files in the project that contain this string.  Note that the Analysis Window shows specifics on the 24 occurrences found in the employee.spb file.  Clicking on any of these lines will highlight its position in the Editing Window, above. 

    Code Review

    Code reviews are an important part of many shops process before putting their code into production.  The goal is to typically to visually review the code for any obvious flaws.  Formatter Plus helps automate this process by performing over 130 checks on your code to identify potential coding errors as well as visual aspects of the code that impact maintainability and readability.

  1. First, place your cursor over the Salary History folder in the Project Browser Window and right-click.
  2. Click on Code Review. Notice the changes in the Results Window. Your files have now been analyzed for Code Correctness, Maintainability, Efficiency, Readability and Program Structure.
  3. Click on the Code Correctness cell for the 1st file (salhist.spb). The Analysis Window below lists the 3 Code Correctness issues associated with the salhist.spb file.  
  4. Notice there are 2 occurrences of the "Use of Char" recommendation. Right mouse-click on this recommendation and click on Code Review Tip to view its importance. Click the (+) sign to review each occurrence of this issue.
  5. Click on any line number below, and its position will be highlighted in the Editing Window, above.

Syntax-Checking

Let's next take a look at how Formatter Plus performs fast offline syntax checks.  With Formatter Plus you don't need to be connected to Oracle to find syntax mistakes. Our example shows finding a simple typo, but complex errors can be isolated just as fast -- and Formatter Plus always supplies insights on what it was expecting to find to help you fix it, too!

  1. In the Project Window, Right mouse-click on the main project folder (Sample Project) and then click Check Syntax. 
  2. All of the files in the sample project will be checked for syntax errors with their status displayed in the Results Window.  Notice that all files except loc.sps are error free.  
  3. Double-click on the loc.sps file in the Results Window.  The error (a space in the word 'cursor') will be highlighted.  In the Editing Window, correct the error and run the Syntax Check again.  Note:  You will not be able to correct this error in the DEMO version.

    Code Profiling

    Lastly, let's get a profile of our entire sample application - number of statements, version dependencies, and dozens of other essential statistics.

  1. In the Project Browser Window, highlight the main project folder (Sample Project) and right-click on it.  Click on Code Profiler. Your folder is now instantly analyzed for 17 different statistical categories. 
  2. Click on each of the Code Profiler categories in the lower-left window for statistics. 
  3. Click the Report link on the Code Profiler Window Bar for a readout of all Code Profiling statistics.  To print this report, from the menu bar select File and then Print.

This officially concludes the "5-Minute Walk-Through". But, did you know that you can perform all of these same actions on your own files?  This next section quickly takes you through formatting and reviewing your personal files.

  1. First, from the menu bar select File and then Open.  Select the file you wish to open and click Open.  You may want to open a file that contains particularly difficult code to read.
  2. In the Editing Window, click the right-mouse button and select Format Text.  The formatted file will be displayed in the Results Window.  
  3. From the menu bar select File and then Print to print a copy of the formatted code.  
  4. From the menu bar select Text Editor, Edit Functions, Revert to return the code sample to it's original state.  Print this out so you can compare it to the Formatted version!
  5. In the Editing Window, highlight any word(s) and click the binocular icon on the toolbar to find the next occurrence of the selection in your file.
  6. In the Editing Window, click the right-mouse button.  Notice the options for Check Syntax, Code Review, and Code Profiler.  Selecting each of these options will perform the associated task on your file.  Make sure to note the output in the Results Window!

DEMO Limitations

An important consideration when working with your DEMO copy of Formatter Plus is that it is a demo.  As such, you can use it try  will demonstrate the capabilities of Formatter Plus, but you will not have the full capabilities of a licensed version.  

In the DEMO version you CANNOT:

  • Create a project
    A sample project has been supplied for demonstrating projects organization
  • Modify the sample project
    This will help ensure that you always have the files you need for the walkthru and quick tour
  • Edit any files in the sample project
    You can however demo Formatter Plus' editing capabilities by opening one of your own files.  Note that any changes you make cannot be saved.
  • Create / Edit a new file
    You cannot create a new file in the edit window or add a new file to the sample project, but definitely open your own files and try Formatter Plus' tools!
  • Format a project
    This function is disabled because the demo version does not allow any file changes to be saved.  Formatting from the project menu saves each file to disk after formatting.
  • View line items in the Code Profile and Code Review
    This feature is only disabled for your own files.  You can test this feature on any files in the sample project.
  • Retain GUI settings on exit
    The demo version always resets the windows to their default position.  The licensed version will remember your window settings on exit for your next execution.
