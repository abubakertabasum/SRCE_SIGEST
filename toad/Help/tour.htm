This section of the User's Guide will provide a "Quick Tour" through Formatter Plus.  You may find it handy to have a hard copy of this page while going through the quick tour.  The quick tour is divided into two sections.  Section One covers actions involving an entire Project and Section Two details actions involving individual Files.  This quick tour will be referencing the files in and the entire project called Sample Project.  

If you would prefer, you may view the Formatter Plus "5-Minute Walk-Through", which will instantly familiarize you with the key features and benefits of this powerful product from Quest Software. Click here to go to the "5-Minute Walk-Through", or continue below to proceed with the Quick Tour.

Section One

Opening a Project

From the Menu Bar, click on Project and then Open. A pop-up dialog box will appear.  Select the project you wish to open and then click Open.  (All formatter projects will be saved using the file extension .rnp).  Open the project titled Sample Project.  By default, it is located in the following folder:  C:\Program Files\RevealNet\Formatter Plus\Sample Code .

The method listed above is for opening an existing project.  For detailed information on Creating a Project, Click Here.

Syntax Checking Projects

To Syntax Check an entire project, choose one of the methods below.  In each case, the Project Name in the Project Bar must be selected.  In this tour, click on Sample Project in the project bar to select it. 

  1. From the Menu Bar, click on Project and then Check Syntax. 
  2. Right click in the Project Bar and select Check Syntax.

Note:  There are 2 other methods that could have been used to accomplish this same task:  clicking on the syntax check icon on the toolbar and selecting the project scope, or right-mouse clicking on the project name and selecting syntax check from the menu.

The results of the syntax check are displayed in the Output Window.  If a file contains an error, the words "Syntax Error" will appear in the column next to the file name.  If the file is error-free, the words "No Errors Found" will appear in the column next to the file name. 

You will notice that one of the files in this project (loc.sps) contains a syntax error.  In the output window, click on loc.sps.  You will see detailed information about the location and cause of the error, as well as suggestions for correcting the error in the lower portion of the output window.

Formatting Options

There are two ways to set formatting options in Formatter Plus.

  1. From the menu bar select Options and then Formatting Options .  This activates a Pop-up dialog box allowing you to set Formatting Preferences in a variety of categories.  You can choose to save the options when you exit the dialog box.  They will then be applied to everything you format until you change the settings again.

  2. Right-mouse click anywhere in the Editing window.  Choose Formatting Options.  The same dialog mentioned in the previous method appears.

Once formatting options have been set, they can be re-set to original factory settings.  In the Formatting Options dialog, highlight the option you would like to reset to the original settings and click the Reset Icon.  Please note, to reset ALL options to their original settings, delete the FmtPlus.opt file found in the Formatter Plus directory on your PC (by default C:\Program Files\RevealNet\Formatter Plus\FmtPlus.opt).  

Formatting a Project

  Note:  This functionality is disabled in the DEMO version.

There are several ways to format an entire project.  Each of these methods is listed below. The project name (in this example, Sample Project) must be selected in the Project Window in order to format the entire project.

  1. From the Menu Bar, select Project then Format Files .

  2. Right-mouse click in the Project Window and select Format Files.

  3. Click the Format Icon  in the Formatting Toolbar or from the right mouse menu in the edit window.  In this case a dialog box appears asking if you would like to format the current editor text or the selected project files.  You can adjust the scope for formatting.  Note:  This box only appears if you choose the format option from the Formatting toolbar or the right mouse options in the edit window.

If a file contains a syntax error, it will not be formatted.  There will be two columns in the Output window when formatting projects.  The first column will list file names.  The second column will tell you if the file has been formatted or not.  If it has, it will say "Formatted".  If an error occurred in the code syntax, the second column will say "Unrecognized Construct or PL/SQL Syntax Error".  Scroll through the list and you will see "Unrecognized Construct or PL/SQL Syntax Error" next to loc.sps.  Click on loc.sps and you will see the lower portion of the output window change to show the syntax error information.  Once the error has been corrected, the file can be formatted.

Code Review and Code Profiler

Formatter Plus includes Code Profiler and Code Review.  The Code Profiler provides a statistical analysis of the main components of PL/SQL applications in the code files.  There are several ways to run the Code Profiler on an entire project.  In each case, the project name (Sample Project) must be selected. 

  1. From the menu bar select Project, then Code Profiler. 

  2. Right-mouse click in the Project Window and click on Code Profiler.

  3. Click the Code Profiler Icon  from the Formatting Toolbar or the Right Mouse Menu in the Edit Window.  In this case, a dialog box will appear asking you to choose the Scope of the code profiler - either profile the current edit text or the selected project file.  Note: This option only occurs when choosing the Code Profiler Icon form the Formatting Toolbar or the Right Mouse menu in the Edit Window.

Clicking on Code Profiler will run a report of the components in the code files contained within the project.  The results will be displayed in the upper portion of the Output window.  Clicking on an item in the grid will display details in the lower section of the Output window.  Double clicking on an item in the lower portion of the Output Window will open the file containing that information in the Edit window.  Clicking on a category in the list to the left of the Output window will display results for that particular topic. Follow the steps below to run the Code Profiler against Sample Project.

  1. From the menu bar select Project, then Code Profiler.

  2. In the Output Window, click on 111 under the heading Decls.  Notice that additional information is displayed in the lower section of the output window.

  3. Double click on the file name (employee.spb) next to the 111 in the Output Window. Notice that the file opens in the edit window.  

  4. Click on Cursor Analysis in the sidebar portion of the Output Window.  Notice the information in the lower portion of the Output window change to show the Cursor Analysis information.

Code Review offers tips and recommendations for improving your code.  You can run the Code Review in the following ways.  Once again, make sure the Project Name (Sample Project) is selected in order to get results on the entire Project.

  1. From the menu bar select Project, then Code Review.

  2. Right Mouse click in the Project Window, and click on  Code Review  .

  3. Click the Code Review Icon from the Formatting Toolbar or the Right Mouse Menu in the Edit Window.  In this case, a dialog box will appear asking you to choose the Scope of the code review - either run the review against the current edit text or the selected project file.  Note: This option only occurs when choosing the Code Review Icon form the Formatting Toolbar or the Right Mouse menu in the Edit Window.

A report grid will be displayed in the Output window.  Clicking on an item in the grid will bring up details in the lower portion of the Output window.  Double clicking on an item in the lower portion of the output window will open/make active the file containing that information in the Edit window.  Holding the mouse on a recommendation in the lower part of the Output window displays a pop up box containing more information about the recommendation.  Right mouse clicking on an item produces an option allowing the user to search the Knowledge Base for Active PL/SQL Development for related information.  Clicking on a category in the list to the left of the Output window will display results for that particular topic.  Follow the steps below to run the Code Review.

  1. From the menu bar select Project, then Code Review.

  2. In the Output bar click on 18 under the heading Code Correctness.  You will notice that the information in the lower portion of the output bar is detailed information about the employee.spb file.

  3. Click on 24 under the heading Maintainability.  

  4. In the lower portion of the output window, Right Mouse click over the line that says "Cursor reference to a external variable (use a parameter)".  Notice the options for Code Review Tip and searching the Knowledge Base for Active PL/SQL Development for more information.

  5. Click on the Code Review Tip to view more information on the recommendation.

  6. Expand the same option.  Click on the line that says "Ln 457, Col 14:  FROM employee".  Notice that the line of code referenced will be highlighted in yellow and marked with an arrow in the edit window. 

  7. Click on Readability in the side bar of the Output window.  Notice the information in the Output window change to reflect the Readability Code Review results.

Section Two

This section will provide information on opening, creating, formatting, syntax checking and editing individual code files.  These procedures are much the same as working with an entire project.

Opening a Single File

The methods for opening an individual file are listed below.

  1. From the menu bar Click on File and then Open.  A pop-up dialog box will appear allowing you to choose the file you wish to open.  Select the file you wish to open and click Open.  The file will open in the Edit Window.  For this tour, open department.sps.  By default, the location of this file is C:\Program Files\RevealNet\Formatter Plus\Sample Files.   

  2. Click the Open Icon on the Standard Toolbar.  The same dialog box as in method one will appear.  Select department.sps and click Open.

Creating a New File

There are two ways to create a new file in Formatter Plus.

  1. From the menu bar click on File and then New.  A blank file will appear in the edit window.  You can then create or insert your code by using the edit functions on the Standard toolbar.

  2. Click the New Icon on the Standard Toolbar.  This will also cause a blank file to appear in the edit window.

When you are done editing a file, click the Save Icon to save your work.

Syntax Checking a File

There are several ways to syntax check an individual file.  In all cases the results will be displayed the same as they are for Projects.  Click Here for information on Syntax Checking a Project.

  1. From the Menu Bar, click Text Editor and then Syntax Check.

  2. From the Formatting Toolbar click on the Syntax Check Icon .

  3. Right-mouse click in the Edit window and select Syntax Check.

Try syntax checking the file department.sps using any of the methods listed above.  The results will be displayed in the Output bar.  The lower portion of the output bar will display detailed information about the location of the error and recommendations for correcting the error.  If you do not see a two part Output Window, click on "Show Details" in the top of the Output Window.

  If there are no errors in syntax, a green indicator and the words "No Errors" will appear in the status bar at the bottom of the screen.   

If errors are present, a Red indicator and the words "Syntax Errors" will appear in the status bar.    The line of code containing the error will be highlighted in yellow and marked with a yellow arrow .   

Formatting a File

There are several ways to format an individual file.  In all cases, the results will be displayed in the same manner as they are for Projects.  Click Here for information on Formatting a Project.

  1. From the Menu Bar, click Text Editor and then Format.

  2. From the Formatting Toolbar click on the Format Icon .

  3. Right-mouse click in the Edit window and select Format.

Try formatting the file department.sps using any of the methods listed above.  The results will be displayed in the Output bar.

Formatting Options

Formatting options will be applied to individual files as well as entire projects.  Try the following exercise for changing formatting options on the department.sps file.

  1. Right-mouse click in the Edit window and select Formatting Options .  Note: You can also select Formatting Options from the Options menu on the Menu bar.

  2. Expand "Formatter Plus Options".

  3. Expand "Ab Case".

  4. Click on "Keywords".

  5. Change "Keywords" from Uppercase to Lowercase.

  6. Click on the Save Icon and Close the Formatting Options dialog box.

  7. Re-format the code.  Notice the keywords changed from UPPERCASE to lowercase.

Code Review and Code Profiling

The Code Profiler and Code Review work the same way for individual files as they do for projects.  Click Here for information on the Code Profiler and Code Review with regard to Projects.

You can activate the Code Profiler for Individual files by using any of the following methods.

  1. From the Menu Bar click Text Editor and then Code Profiler.

  2. On the Formatting Toolbar click the Code Profiler Icon .

  3. Right-mouse click in the Edit window and select Code Profiler.

You can activate the Code Review for individual files by using any of the following methods.

  1. From the Menu Bar click Text Editor and then Code Review.

  2. On the Formatting toolbar click the Code Review Icon .

  3. Right-mouse click in the Edit window and select Code Review.

Try the following exercise on the department.sps file.

  1. Click the Code Profiler Icon on the formatting toolbar.

  2. In the list to the left of the Output Window, click .  You may have to scroll down the list to see this option.

  3. In the top portion of the output window click in the box below the header "Func" (it contains the number 23).  This will highlight and expand Functions in the lower section of the project window.

  4. In the lower part of the project window, double click on the following:
    Ln40: function isnullpky(rec_in in department%rowtype)  In the edit window, this line will be marked with an arrow and highlighted in yellow.

  5. Try clicking on some of the other values in the Code Profiler  and Code Review sections.
